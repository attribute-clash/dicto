# Firmware (ESP32-C3) для диктофона с одной кнопкой

Эта прошивка реализует базовый рабочий сценарий из ТЗ:

- удержание кнопки → запись;
- отпускание кнопки → остановка;
- сохранение WAV на SD;
- периодическая отправка неотправленных файлов на сервер по Wi‑Fi;
- пометка отправленных файлов через `.sent` marker.

## Быстрый старт

1. Установите [PlatformIO](https://platformio.org/).
2. Откройте папку `firmware` как проект PlatformIO.
3. Отредактируйте compile-time параметры в `src/main.cpp`:
   - `WIFI_SSID`, `WIFI_PASSWORD`
   - `SERVER_URL`
   - `API_TOKEN`
   - `DEVICE_ID`
4. При необходимости поменяйте pin mapping (`PIN_BUTTON`, `PIN_LED`, `PIN_SD_CS`, `PIN_I2S_*`).
5. Скомпилируйте и прошейте устройство.

## Описание протокола отправки

Сейчас отправка делается как `POST` c телом `audio/wav`.

Дополнительные метаданные идут HTTP-заголовками:

- `Authorization: Bearer <token>`
- `X-Device-Id: ...`
- `X-Recorded-At: ...`

Если вашему серверу нужен `multipart/form-data`, замените функцию `uploadFile()`.

## Структура на SD

- `/audio/*.wav` — записанные файлы;
- `/audio/*.wav.sent` — маркер успешной отправки.

## Ограничения текущей версии

- Для HTTPS используется `setInsecure()` (нужно заменить на pinning/CA для production).
- Нет циклической очистки SD при заполнении.
- Нет отдельного энергосберегающего режима deep sleep.
