# Эмуляция прошивки на ПК (микрофон + пробел)

Этот модуль позволяет воспроизвести поведение устройства на ПК:

- зажали `SPACE` → начинается запись с микрофона;
- отпустили `SPACE` → запись завершается и сохраняется в WAV;
- в фоне неотправленные WAV отправляются на сервер распознавания;
- после успеха создаётся `.sent` marker, повторно файл не отправляется.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r emulator/requirements.txt
```

## Запуск эмулятора

```bash
DEVICE_ID=DCT-001-PC \
SERVER_URL=http://127.0.0.1:8080/api/v1/audio/upload \
API_TOKEN=dev-token \
python -m emulator.pc_dictaphone
```

Управление:
- удерживайте `SPACE` для записи;
- `ESC` — выход.

## Какие есть варианты эмуляции

### Вариант 1 (реализован): realtime push-to-talk с клавишей SPACE
- Максимально близко к железу: «кнопка зажата — запись идёт».
- Подходит для ручного E2E тестирования с живой речью.

### Вариант 2 (рекомендуется добавить при необходимости): режим "tap-to-record"
- Нажали `R` — старт, `R` — стоп.
- Удобно, если ОС/драйверы плохо обрабатывают hold-space события.

### Вариант 3 (рекомендуется добавить): playback-эмулятор без микрофона
- Берёт заранее подготовленные WAV из папки и отправляет по расписанию.
- Полезно для CI и нагрузочных тестов очереди/ретраев без звукового железа.

### Вариант 4 (рекомендуется добавить): loopback через виртуальный аудиодрайвер
- На вход эмулятора подаётся системный output (например, TTS или звонок).
- Полезно для регрессионных тестов ASR качества на фиксированном наборе аудио.

## Контракт отправки

Текущий Python-эмулятор отправляет `POST` с телом `audio/wav` и заголовками:

- `Authorization: Bearer <token>`
- `Content-Type: audio/wav`
- `X-Device-Id: ...`
- `X-Recorded-At: ...`

Если сервер ожидает `multipart/form-data`, измените функцию `upload_file()`.
